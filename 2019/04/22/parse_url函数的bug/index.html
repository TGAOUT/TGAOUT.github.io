parse_url函数的bug

发表于 2019-04-22 | 分类于 php函数

前言

最近打某个ctf比赛，接触到parse_url的一些bug，其实早在2016年就成为ctf中的一个常见考点。

正文

由于parse_url函数在解析url时存在的bug导致在某些情况下可以绕过正则的过滤

用法

parse_url — 解析 URL，返回其组成部分

mixed parse_url ( string $url [, int $component = -1 ] )

bug之///会被返回false

此bug在任何php版本中都适用

来看源代码

尝试：http://127.0.0.1:5555/422.php?payload=flag

返回结果

尝试：127.0.0.1:5555///422.php?payload=flag

返回结果

可以发现parse_url遇到前三个字符为///,会返回false。

bug之参数component

此bug在任何php版本中都适用

如果指定了 component 参数， parse_url() 返回一个 string（或在指定为 PHP_URL_PORT 时返回一个 integer）而不是array。如果 URL 中指定的组成部分不存在，将会返回 NULL。
